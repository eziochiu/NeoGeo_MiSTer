## 三、功能模块详解

### 3.1 顶层模块 `neogeo.sv`

**文件大小：** ~2258 行，是整个核心的"总装车间"

**关键职责：**
- 定义 MiSTer 框架接口（视频、音频、HPS、SDRAM、DDR3）
- 实例化所有子模块并连线
- 处理 OSD 配置（系统类型、视频模式、DIP 开关等）
- 管理 ROM 加载流程（从 HPS 接收数据写入 SDRAM/BRAM）
- 混合音频输出（FM + ADPCM + PSG）

**系统类型支持：**
- `status[2:1] = 00` → Console (AES 家用机)
- `status[2:1] = 01` → Arcade (MVS 街机)
- `status[2:1] = 10` → CD
- `status[2:1] = 11` → CDZ

---

### 3.2 CPU 模块 (`rtl/cpu/`)

#### `cpu_68k.v` — 主 CPU
- **封装** fx68k 开源 IP 核（Jorge Cwik 编写）
- 时钟：12MHz（通过 CLK_EN_68K_P/N 使能信号从系统时钟分频）
- 24 位地址总线，16 位数据总线
- 支持中断（IPL0-2）、总线仲裁（BR/BG/BGACK）

#### `cpu_z80.v` — 声音 CPU
- **封装** T80 开源 IP 核
- 时钟：4MHz
- 负责控制 YM2610 声音芯片
- 与 68000 通过共享寄存器通信

**对应 PGM 的启示：**  
PGM 也用 68000，可以直接复用 fx68k。PGM 没有 Z80，声音由 68000 + ICS2115 处理。

---

### 3.3 视频系统 (`rtl/video/`)

这是最复杂的部分，模拟了 Neo Geo 的多个自定义 ASIC：

| 文件 | 对应芯片 | 功能 |
|------|---------|------|
| `lspc2_a2.v` | LSPC2-A2 | **主视频控制器**（26KB，最大文件）：扫描线时序、精灵/Fix 图层调度、VRAM 访问仲裁 |
| `neo_b1.v` | NEO-B1 | **像素多路选择器**：合并精灵层和 Fix 层，输出最终像素 |
| `zmc2_dot.v` | NEO-ZMC2 | **精灵像素解码器**：从压缩的位平面数据中提取像素 |
| `videosync.v` | — | 同步信号生成（HSYNC/VSYNC/消隐） |
| `linebuffer.v` | — | 行缓冲（双缓冲，一行渲染一行输出） |
| `fast_cycle.v` | — | **快速 VRAM 周期**：精灵数据预取（15.9KB，第二大文件） |
| `slow_cycle.v` | — | **慢速 VRAM 周期**：Fix 图层数据读取 |
| `hshrink.v` | — | 精灵水平缩放 |
| `irq.v` | — | 视频中断控制（VBlank、HBlank、定时器） |
| `lspc_timer.v` | — | 可编程扫描线定时器 |
| `lspc_regs.v` | — | LSPC 寄存器读写 |
| `autoanim.v` | — | 自动动画计数器 |
| `neo_cmc.v` | NEO-CMC | 加密/bankswitch 芯片（部分游戏使用） |

**渲染流水线：**
```
VRAM → LSPC2(调度) → Fast Cycle(精灵预取)
                    → Slow Cycle(Fix预取)
                           ↓
SDRAM(图形数据) → ZMC2(像素解码) → Line Buffer
                                       ↓
                              NEO-B1(图层合并) → 像素输出
```

**对应 PGM 的启示：**  
PGM 的 IGS023 类似 LSPC2 + NEO-B1 的组合，但精灵系统更复杂（支持更多精灵、不同的缩放方式）。

### 3.4 I/O 系统 (`rtl/io/`)

模拟 Neo Geo 主板上的各种自定义芯片：

| 文件 | 对应芯片 | 功能 |
|------|---------|------|
| `neo_c1.v` | NEO-C1 | **系统控制器**：地址解码、等待状态、手柄输入 |
| `neo_d0.v` | NEO-D0 | ROM bankswitch 控制 |
| `neo_e0.v` | NEO-E0 | 地址线混洗（反盗版） |
| `neo_f0.v` | NEO-F0 | 68000/Z80 通信桥 |
| `neo_g0.v` | NEO-G0 | 音量/声道控制 |
| `neo_pvc.v` | NEO-PVC | 高级加密保护芯片（后期游戏） |
| `neo_sma.sv` | NEO-SMA | 加密地址 bankswitch（KOF99 等） |
| `syslatch.v` | — | 系统锁存寄存器 |
| `watchdog.v` | — | 看门狗定时器 |
| `upd4990.v` | μPD4990 | 实时时钟芯片 |
| `z80ctrl.v` | — | Z80 总线控制 |
| `pcm.v` | — | PCM 声音数据通道 |
| `clocks.v` | — | 时钟分频/使能信号生成 |
| `zmc.v` | NEO-ZMC | 地址线转换 |

**对应 PGM 的启示：**  
PGM 的 I/O 相对简单（没有这么多保护芯片变种），但 IGS025/027A 的加密保护比 NEO-SMA/PVC 更复杂。

---

### 3.5 声音系统 (`rtl/jt12/` + `rtl/jt49/`)

#### YM2610 FM 合成 (`jt12/`)
- **Jotego** 编写的 YM2610（jt10）FPGA 实现
- 包含：FM 合成器（4通道）、ADPCM-A（6通道采样）、ADPCM-B（1通道采样）
- 关键文件：
  - `jt10.v` — YM2610 顶层
  - `jt12_op.v` — FM 算子（正弦波 + 包络）
  - `jt12_eg.v` — 包络发生器
  - `jt10_adpcm_drvA.v` — ADPCM-A 驱动（6声道打击乐采样）
  - `jt10_adpcmb.v` — ADPCM-B 驱动（1声道语音采样）
  - `jt12_mixer.v` — 最终混音输出

#### SSG/PSG (`jt49/`)
- YM2149 兼容的可编程声音发生器
- 3 个方波通道 + 1 个噪声通道 + 包络

**对应 PGM 的启示：**  
PGM 使用 ICS2115（波表合成器），与 YM2610 完全不同。ICS2115 需要从零用 Verilog 实现，参考 MAME 的 `ics2115.cpp`。

---

### 3.6 内存系统 (`rtl/mem/`)

| 文件 | 功能 |
|------|------|
| `sdram_mux.sv` | **SDRAM 多路复用器**（核心文件）：仲裁 68k ROM、精灵图形、Fix 图形、DMA 等对 SDRAM 的访问 |
| `sdram.sv` | SDRAM 物理控制器（读/写/刷新时序） |
| `ddram.sv` | DDR3 RAM 接口（通过 HPS 侧） |
| `dpram.v` | 双端口 BRAM（用于片内小容量存储） |
| `backup.v` | 备份 RAM（存档） |
| `memcard.v` | 记忆卡模拟 |

**SDRAM 映射（卡带模式）：**
```
0x0000000 - 0x00FFFFF  P1 ROM      (1MB)   — 68k 程序
0x0200000 - 0x05FFFFF  P2 ROMs     (4MB)   — 68k 程序扩展
0x0600000 - 0x061FFFF  System ROM  (128KB) — BIOS
0x0620000 - 0x063FFFF  SFIX ROM    (128KB) — 系统 Fix 字库
0x0680000 - 0x06FFFFF  S ROM       (512KB) — 游戏 Fix 图形
0x0800000 - 0x1FFFFFF  C ROM       (24MB)  — 精灵图形数据
```

**SDRAM 多路复用策略：**
SDRAM 运行在 120MHz（5×24MHz），在一帧时间内轮流服务：
1. 68000 程序 ROM 读取（优先级最高）
2. 精灵图形数据读取（PCK1B 触发）
3. Fix 图形数据读取（PCK2B 触发）
4. SDRAM 刷新周期
5. DMA 传输（Neo CD 模式）

**对应 PGM 的启示：**  
PGM 的 SDRAM 映射需要重新设计，因为 PGM 的 ROM 更大（P ROM 可达 4MB，精灵数据可达 32MB+），可能需要双 SDRAM 或 DDR3。

## 五、关键实现细节

### 5.1 时钟体系
- 系统主时钟：24MHz（原始 Neo Geo）
- SDRAM 时钟：120MHz（5×24MHz）
- 68000 时钟：12MHz（24/2），通过使能信号实现
- Z80 时钟：4MHz（24/6），通过使能信号实现
- 像素时钟：6MHz

**实现方式：** 不是用真正的分频时钟，而是在高频时钟（如 96MHz）下用 `CLK_EN` 使能信号来"模拟"低频时钟边沿。这样所有模块共享同一个时钟域，避免跨时钟域问题。

### 5.2 SDRAM 访问策略
SDRAM 是整个系统的瓶颈——所有 ROM 数据都在里面。`sdram_mux.sv` 实现了一个**优先级轮询调度器**：
1. 利用 120MHz 的高速时钟，在不同的时间槽服务不同请求
2. 精灵数据用 4-word burst 读取（一次读 64 位 = 16 像素）
3. Fix 数据用单次 16 位读取（4 像素）
4. 数据在加载时被重排（reorder），这样运行时可以直接顺序读取

### 5.3 图形数据重排
ROM 加载时，精灵和 Fix 的位平面数据被重新排列：
- **精灵：** 原始 C ROM 的奇偶 ROM 交织存储，加载时重排为连续的 4 位平面
- **Fix：** 列优先存储改为行优先，匹配 16 位 SDRAM 总线宽度

### 5.4 保护芯片
Neo Geo 后期游戏使用了多种保护方案：
- **NEO-SMA** (`neo_sma.sv`)：地址线加密 bankswitch（KOF99、Garou 等）
- **NEO-PVC** (`neo_pvc.v`)：数据加密（KOF2003、SvC Chaos 等）
- **NEO-CMC** (`neo_cmc.v`)：Fix 图形加密 + bankswitch

## 六、如果要做 PGM 核心，需要对应实现什么

| Neo Geo 模块 | PGM 对应 | 复用可能性 |
|-------------|---------|-----------|
| fx68k (68000 CPU) | 同 68000 | ✅ 直接复用 |
| T80 (Z80 CPU) | 无 | ❌ PGM 没有 Z80 |
| jt10 (YM2610) | ICS2115 | ❌ 需要从零实现 |
| LSPC2 (视频控制) | IGS023 | ❌ 需要从零实现，但架构可参考 |
| NEO-B1 (像素合并) | IGS023 的一部分 | ❌ 需要从零实现 |
| NEO-C1 (I/O) | PGM I/O | ⚠️ 需要重写，但更简单 |
| NEO-SMA/PVC (保护) | IGS025/027A | ❌ 完全不同的加密方案 |
| sdram_mux (内存仲裁) | 类似但需重新设计 | ⚠️ 架构可参考，映射需重做 |
| MiSTer sys/ 框架 | 完全相同 | ✅ 直接复用 |
